# Why use it? (The "Professor" Use Cases)

1. Code Reviews without Branches:
   If you want to show your friend a "concept"
   for the Prisma schema
   without cluttering your Git history with a new branch.

2. Fixing "Old" Bugs:
   You find a bug in an old version of the app.
   You can create a patch and apply it
   to both the old version and the current version.

3. No Internet / Air-gapped Systems:
   If you’re working on your Arch machine without a network,
   you can still "hand over" code via a patch file.
