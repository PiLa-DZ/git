# GitHub Discussions --> Q&A, and general conversation

## 1. Enabling Discussions

1. Navigate to your repository on GitHub.
2. Click on the Settings tab (the gear icon).
3. Scroll down to the Features section.
4. Find the checkbox labeled Discussions and check it.
5. GitHub will immediately add a "Discussions" tab to your repository's top navigation bar.

## 2. Discussion Categories

- Announcements: Important updates (e.g., "We are switching from MariaDB to PostgreSQL").
- General: Random thoughts or "check-ins."
- Ideas: Brainstorming new features (e.g., "Should we add a dark mode for the student portal?").
- Q&A: Asking for technical help without it being a "bug."
- Show and Tell: Showing off a cool CLI tool you found for Arch that helps with Node.js debugging.

## 3. Key Social Features

- Upvoting: most popular thoughts rise to the top.
- Mark as Answer: once a question is solved, you can mark a specific comment as the "Answer."
- Threaded Replies: reply directly to a specific person's comment, creating a sub-thread.

## 4. Converting to an Issue

One of the most powerful workflows is starting a debate in Discussions
and, once you’ve decided exactly how to build the feature,
clicking the "Create issue from discussion" button.

This copies all the context, links,
and ideas into a brand new Issue
so you can start coding in Neovim with a clear plan.
