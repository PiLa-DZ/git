- PR from a Fork
  1. Fork: -----------> (your server-side copy)
  2. Clone: ----------> Bring your fork down to your Arch machine
  3. Branch: ---------> Create a new feature branch locally
     - Upstream ---------> you should link your local to the original project (usually called upstream)
  4. Commit & Push: --> Make your changes, then push them back to your fork
  5. Open PR: --------> On the original repository's GitHub page
     - checkbox ---------> that says "Allow edits from maintainers --> Keep this checked It allows the original owner to make tiny fixes
  6. Review: ---------> The original maintainer reviews your code
