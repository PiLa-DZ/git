# GitHub Essentials

---

## 1. Creating Your Account

1. Go to github.com.

2. Tip: Use a professional email (or the one you used in your git config --global user.email).

3. Username: Choose something that will look good on a resume (e.g., dev-yourname or arch-coder).

4. Verification: Complete the "Spiral Galaxy" puzzle to prove you aren't a bot.

---

## 2. The GitHub Interface

The dashboard can be overwhelming at first.
Here are the "Hotspots" you'll use daily:

- Top Search Bar: Find any open-source project (like the Linux Kernel or Express.js).

- The "+" Dropdown: Your "Command Center" for creating new Repositories or Gists.

- The "Pull Requests" Tab: See all the code changes people are asking you to review.

- The "Issues" Tab: A "To-Do List" for bugs and feature requests.

---

## 3. Setting Up Your Profile

Your profile is your Developer Portfolio.

### The Profile README (The "Secret" Feature)

You can create a special README that shows up on your main profile page.

1. Create a new repository
   with the exact same name as your username
   (e.g., if your username is linux-fan, the repo is linux-fan).

2. Initialize it with a README.md.

3. GitHub will show a "special" badge saying you found a secret.

4. Content:
   Use this to list your skills
   (Node.js, Arch Linux, Git),
   your current projects, and your social links.

---

## 4. Creating Repositories

When you click "New Repository,"
you’ll see several options that define how the world sees your code.

### Public vs. Private: Which should you choose?

| Type    | Access                                       | Best For...                                                            |
| ------- | -------------------------------------------- | ---------------------------------------------------------------------- |
| Public  | Anyone on the internet can see your code.    | "Open Source, Portfolios, and Learning."                               |
| Private | Only you (and people you invite) can see it. | "Client work, projects with sensitive logic, or ""Work in Progress.""" |

---

## Essential Setup Options

- Add a README file:
  Always do this.
  It’s the "Front Door" of your project.

- Add .gitignore:
  GitHub provides templates.
  Pick the "Node" template to automatically ignore node_modules.

- Choose a License:
  For public projects,
  the MIT License is the most popular
  it basically says "Use this however you want, just don't sue me."
