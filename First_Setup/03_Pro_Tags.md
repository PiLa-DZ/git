# Pro Tip: Semantic Versioning (SemVer)

When tagging your "students_courses_app",
follow the vMAJOR.MINOR.PATCH rule:

- Major: Breaking changes (e.g., changed the whole database schema).

- Minor: New features (e.g., added a new API endpoint).

- Patch: Bug fixes (e.g., fixed a typo in the login logic).
